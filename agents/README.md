# Agents Directory

This directory contains reusable operating briefs for the project team.

- `lead-codex-agent.md`
- `research-agent.md`
- `data-agent.md`
- `hardcore-developer.md`
- `designer.md`
- `software-tester.md`

Use these briefs as lenses during planning and execution. For example, a feature
request can be reviewed by the Research Agent for novelty, the Designer for
experience, the Data Agent for signal design, the Hardcore Developer for
implementation, and the Software Tester for validation.

## Invocation Pattern

When we want a specific perspective, call it explicitly:

- `Research Agent:` unconventional signals, references, risks, and hypotheses.
- `Data Agent:` metrics, schemas, datasets, evidence loops, and validation.
- `Designer:` product feel, UX flows, visual systems, and interaction ideas.
- `Hardcore Developer:` architecture, implementation, code, and verification.
- `Software Tester:` edge cases, test plans, bugs, and regression risk.
- `Lead Codex Agent:` synthesis, tradeoffs, sequencing, and next actions.
